# Stop & Lock 777 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add **Stop & Lock 777** — a player-stopped hold-and-spin cash-collect machine — as the floor's Featured machine, in a new `lock-reel` engine family.

**Architecture:** New engine family `lock-reel` mirroring how `blackjack-reel` (Flameout 21) was added: a pure-data `LockReelMachineDef`, an interactive session state, step functions (`dealStart` / `stopReel` / `resolve` / `bonusStop`), an exact-RTP module (base enumeration + bonus respin-feature EV), store actions, a bespoke full-page cabinet UI, and X-ray / PAR surfaces. RTP is computed (never asserted) and cross-checked by `pnpm verify`. Every cash value is whole-credit, so no fractional-payout departure is needed.

**Tech Stack:** Nuxt 4 / Vue 3 `<script setup>` / Pinia (options store) / TypeScript strict / Vitest (happy-dom). `@stylistic` lint (no semicolons, single quotes, no trailing commas, one statement/line; Vue one attribute per line). Spec: `docs/superpowers/specs/2026-06-18-stop-and-lock-777-design.md`. Approved mockup: `.superpowers/brainstorm/4865-1781794331/content/vault-v4-bigdaddy.html`.

**Per-task convention:** each task ends with `pnpm lint && pnpm typecheck && pnpm test` (and `pnpm verify` once the def exists) green before committing. **Commits off-hours** (outside M–F 7am–7pm), **no AI co-author trailer**, **do not push** until the owner says so. Work on a feature branch (create via `superpowers:using-git-worktrees` at execution start). Confined typecheck errors are acceptable mid-task only between Tasks 1–2 (the union isn't fully wired until Task 2).

**The mechanic (from the spec):** 5 reels / 5×4 grid; reels spin nonstop; STOP locks a reel's 4-cell column via an honest uniform window draw; collect = bet-scaled sum of locked cash + prizes; three 7s locked in one stop-through → free-spin hold-and-spin bonus immediately; two 7s → one re-stop tease. Target RTP ~94–95%, high variance.

```ts
export interface LockReelMachineDef {
  id: string
  name: string
  family: 'lock-reel'
  denominationCents: number
  maxCoins: number
  history: string
  rows: number // grid rows (4)
  reels: SymbolId[][] // five strips of symbol ids (cash ids, prize ids, SEVEN, BLANK)
  symbols: Record<string, { label: string, icon?: string }>
  cashValues: Record<string, number> // symbol id -> credit value per coin (e.g. CASH25 -> 25)
  prizes: Record<string, number> // MINI/MAJOR/GRAND -> credit value per coin
  sevenSymbol: SymbolId // 'SEVEN'
  blankSymbol: SymbolId // 'BLANK'
  bonus: { respins: number, sevenUpgrade: number, grandOnFill: string } // grandOnFill = prize id
  progressive: null
}

export interface LockReelSessionState {
  phase: 'idle' | 'spinning' | 'bonus' | 'resolved'
  grid: (SymbolId | null)[][] // [reel][row], null = unstopped/empty
  idx: number // reels stopped (0..5)
  sevenCount: number
  collectCredits: number // running locked-cash credits at the current bet
  respinsLeft: number // bonus
  ante: number // coins
}
```

- [ ] **Step 3: Implement `app/engine/lockReel.ts`.** Honest stop = uniform draw of a contiguous `rows`-length window from `reels[idx]` (`start = floor(rand()*len)`, wrap). Lock that column into `grid[idx]`; add `cashValues`/`prizes` credits to `collectCredits` (scaled by ante at book time, kept per-coin here); increment `sevenCount` per `sevenSymbol`. `resolve`: on the 5th stop, if `sevenCount >= 3` → `phase:'bonus'`, `respinsLeft = bonus.respins`; else `phase:'resolved'` (the SpinOutcome's `totalPayout` = the collect). `bonusStop`: re-spin empty cells, stop locks new symbols, reset `respinsLeft` on any new lock, sticky 7s add `bonus.sevenUpgrade`; grid full → add `prizes[bonus.grandOnFill]`; `respinsLeft==0` → resolve with the full collect.
